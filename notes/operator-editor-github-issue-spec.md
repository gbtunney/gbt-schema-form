# Operator Editor – GitHub Native Issue Spec

Use this document to create GitHub issues for the Operator Editor repo.

Create these labels first if they do not already exist:

- `ui`
- `ux`
- `feature`
- `bug`
- `architecture`
- `schema`
- `evidence`
- `proposal`
- `history`
- `attachments`
- `dev-env`
- `storybook`
- `api`

Each numbered section in the full spec should become a GitHub issue.
